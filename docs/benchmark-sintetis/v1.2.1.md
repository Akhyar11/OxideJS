## Perubahan Penting Dibanding Versi Sebelumnya

- Layer `RNN` sekarang memakai penamaan bias `bh` yang konsisten dengan hidden state, tetapi tetap kompatibel dengan format serialisasi lama lewat fallback `by` saat `load()`.
- `RNN.save()` dan `RNN.load()` sekarang ikut menyimpan serta memulihkan `hStateful`, sehingga model `stateful` bisa di-serialisasi tanpa kehilangan hidden state terakhir.
- `RNN.forward()` sekarang lebih ketat: `returnState=true` ditolak eksplisit karena belum didukung, input row harus cocok dengan `units`, dan sequence kosong (`seqLen < 1`) langsung dianggap invalid.
- `RNN` memperbarui `inputShape` dan `outputShape` berdasarkan sequence aktual, sehingga shape runtime tidak lagi terkunci ke default konstruktor.
- `Sequential.fit()` sekarang menambahkan guard untuk semua layer recurrent (`RNN`, `LSTM`, `GRU`): training dipaksa `batchSize=1`, sample sequence harus memiliki panjang minimal 1, dan konfigurasi `stateful=true` tidak boleh dipakai bersama `shuffle=true`.
- Versi ini juga menambahkan correctness contract khusus `RNN` untuk shape forward/backward, state carry/reset, save/load roundtrip, penolakan `returnState`, dan validasi aturan fit recurrent.

## Rincian Layer RNN

- Persistensi state: hidden state terakhir sekarang menjadi bagian dari artefak save/load, jadi perilaku `stateful` lebih deterministik setelah model disimpan lalu dimuat kembali.
- Backward/update parameter: penamaan gradien bias ikut dirapikan dari `dBy` menjadi `dBh`, mengikuti perubahan nama parameter bias di layer.
- Validasi API: alih-alih diam-diam menerima mode yang belum siap, `RNN` sekarang fail fast ketika dipakai dengan kombinasi input atau opsi yang tidak valid.
- Kompatibilitas: perubahan format bias tidak memutus model lama karena loader masih menerima field legacy `by`.

## Interpretasi

- Throughput `transformers` tetap tertinggi pada workload ini.
- `RNN` masih menjadi recurrent baseline tercepat, diikuti `GRU`, lalu `LSTM`.
- Nilai utama `v1.2.1` bukan di kenaikan throughput, tetapi di peningkatan correctness dan keamanan pemakaian layer recurrent, terutama `RNN`.
- Guard baru di `Sequential.fit()` mengurangi risiko training recurrent yang keliru tetapi sebelumnya tetap lolos, misalnya batching generik untuk sequence atau kebocoran state saat `shuffle`.

## Tindak Lanjut

- Jika `returnState` ingin didukung penuh, perlu ada desain output API yang jelas untuk membedakan tensor output dan hidden state akhir.
- Guard recurrent di `Sequential` saat ini masih bersifat defensif; bila nanti ingin mendukung batching sequence yang benar, perlu representasi batch khusus selain skema kolom matrix generik.
- Pertahankan correctness contract `RNN` setiap kali ada refactor recurrent berikutnya agar perubahan performa tidak merusak perilaku layer.
