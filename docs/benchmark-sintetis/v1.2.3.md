# Benchmark Sintetis `v1.2.3`

## Metadata

- Tanggal: 22 April 2026
- Basis versi: `1.2.3`
- Branch: `develop-mode`
- Tujuan snapshot: Mengukur ulang throughput recurrent setelah benchmark utama tidak lagi memakai loop sample-per-sample sebagai jalur training utama.

## Perubahan Fairness

- Benchmark recurrent lama pada `v1.2.2` memproses batch efektif dengan loop sample-per-sample.
- Snapshot ini memakai:
  - `Embedding.forwardTimeMajor(...)`
  - `RNN.forwardBatch(...)` / `backwardBatch(...)`
  - `LSTM.forwardBatch(...)` / `backwardBatch(...)`
  - `GRU.forwardBatch(...)` / `backwardBatch(...)`
- Artinya recurrent sekarang diukur dengan batch sequence yang valid dan lebih dekat ke kapasitas implementasi sebenarnya.

## Correctness Companion

- Status correctness recurrent: `pass`
- Entry correctness: `test/correctness/index.ts`
- Entry suite gabungan: `test/index.ts`
- Cakupan correctness yang menyertai snapshot ini:
  - forward shape `returnSequences=false/true`
  - backward shape
  - `stateful` carry + `resetState()`
  - save/load roundtrip
  - `returnState` fail-fast
  - misuse guard di `Sequential.fit()`
  - pipeline `Embedding -> recurrent -> Dense`
  - validasi load payload malformed untuk `LSTM` dan `GRU`
  - smoke coverage `GRU` bidirectional

## Interpretasi Singkat

- Kenaikan terbesar datang dari Tier 1: recurrent sekarang benar-benar memproses batch sequence dalam satu jalur compute time-major yang valid.
- `RNN` menjadi model tercepat pada workload ini, karena overhead gate dan state update paling kecil.
- `GRU` dan `LSTM` juga naik tajam setelah loop sample-per-sample dihapus dari benchmark utama.
- Angka `transformers` relatif stabil, yang menunjukkan optimasi ini terlokalisasi pada family recurrent dan tidak mengorbankan jalur transformer.

## Limitasi

- Jalur batch baru saat ini dipakai oleh harness benchmark recurrent, bukan oleh `Sequential.fit()` generic.
- `stateful=true` pada batched recurrent path tetap dibatasi ke `batchSize=1`.
- `returnState` tetap belum didukung; mode tersebut masih gagal eksplisit.
- Snapshot performa ini didampingi correctness companion untuk recurrent, tetapi belum menambah suite correctness khusus yang menargetkan API `forwardBatch/backwardBatch` secara langsung sebagai public contract terpisah.
