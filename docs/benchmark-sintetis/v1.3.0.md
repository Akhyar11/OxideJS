## Perubahan Penting Dibanding Versi Sebelumnya

- `Transformers.forward()` sekarang mengeluarkan logits seluruh sequence saat mode training, bukan hanya representasi token terakhir.
- `Transformers.backward()` sekarang membangun gradien cross-entropy dari seluruh posisi valid non-pad pada target shifted `[seqLen, batch]`.
- Ditambahkan pemisahan eksplisit antara training path (`forward()` saat train / `forwardFullSequence()`) dan inference path (`forwardNextToken()` / `predict()`).
- Harness benchmark transformer sekarang mengirim target sequence shifted dari window sample yang sama. Jumlah sample tetap, tetapi objective tidak lagi apple-to-apple terhadap `v1.2.3` karena kerja loss per sample bertambah dari satu prediksi menjadi banyak prediksi valid per sample.

## Interpretasi

- Dibanding `v1.2.3`, throughput transformer turun tajam. Baseline sebelumnya untuk transformer adalah `498.30 samples/s`, `126.05 ms/batch`, `2.0068 ms/sample`, sedangkan snapshot ini mencatat `40.74 samples/s`, `1541.83 ms/batch`, `24.5470 ms/sample` pada run transformer terisolasi.
- Penurunan ini diharapkan karena benchmark sekarang benar-benar menghitung objective causal LM full-sequence. Satu sample benchmark memicu jauh lebih banyak logits yang ikut loss dan backward daripada last-token-only path lama.
- Karena objective benchmark berubah, perbandingan terhadap `v1.2.3` hanya semi-comparable: dataset, `seqLen`, `batchSize`, `units`, `heads`, dan jumlah sample tetap sama, tetapi jumlah posisi yang ikut loss per sample tidak sama.
- Entry point benchmark utama repo tetap `npm test`, tetapi angka paling berguna untuk memantau refactor ini adalah benchmark transformer terisolasi karena ia mencantumkan metadata `records/samples/warmup/batches` lengkap.

## Tindak Lanjut

- Jika throughput transformer menjadi prioritas, optimalkan loss/gradient full-sequence agar tidak perlu menyalin probabilitas semua vocab untuk setiap posisi valid di JavaScript.
- Pertimbangkan helper loss native atau sparse masked softmax-cross-entropy agar overhead masking pad dan shifting turun.
- Tambahkan benchmark khusus inference generation agar pemisahan training path vs inference path dapat dipantau terpisah.
- Pertahankan `npm test` sebagai entry point resmi. Jika kelak command benchmark utama berubah, dokumentasikan alasannya secara eksplisit dan pertahankan histori append-only.

## Versioning

Versi aktif proyek saat ini adalah `1.3.0`.

Proyek ini memakai format versi `MAJOR.MINOR.PATCH` seperti `1.3.0`.

- Angka paling depan (`MAJOR`): perubahan besar yang biasanya membawa breaking change atau perubahan arsitektur utama.
- Angka tengah (`MINOR`): penambahan fitur baru atau peningkatan yang tetap kompatibel dengan versi sebelumnya.
- Angka paling belakang (`PATCH`): perbaikan bug, optimasi kecil, cleanup, atau perubahan minor yang tidak mengubah API utama.

Contoh:
- `1.3.0`: rilis mayor `1`, minor `3`, patch `0` untuk perubahan arsitektur training transformer.
- `1.1.4`: masih di mayor `1` dan minor `1`, tetapi sudah ada 4 patch/perbaikan kecil dari baseline `1.1.0`.
