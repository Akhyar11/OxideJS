## Perubahan Penting Dibanding Versi Sebelumnya

- Ditambahkan harness pembanding baru `test/benchmark/transformer_perf_breakdown.ts` untuk memisahkan `forward only`, `backward only`, `training step`, dan `stage profile`.
- `Transformers.backward()` sekarang memakai output dense dari forward pass yang sudah ada, sehingga tidak melakukan `dense.forward(this.xRes2)` kedua kali saat membangun loss gradient.
- Pembangunan gradien loss full-sequence diubah dari nested loop JavaScript per `(validToken, vocab)` menjadi softmax in-place + subtract target + zero invalid columns.
- Objective training, shape output, causal mask, pad masking, dan API publik tidak berubah.

## Interpretasi

- Snapshot `v1.1.6` tidak lagi cocok sebagai pembanding final untuk branch ini karena objective training transformer saat itu berbeda. Ia tetap berguna sebagai histori, tetapi tidak apple-to-apple.
- Benchmark utama yang fair untuk patch ini adalah `transformer_perf_breakdown_before_after`, karena command, config, dan jumlah run identik.
- Hasil before/after menunjukkan patch ini benar-benar membantu, tetapi hanya pada jalur backward/loss. Forward path tidak ikut membaik dan kadang lebih lambat karena noise serta bottleneck baru yang tersisa ada pada softmax/loss gradient full-sequence.
- `pad mask creation` dan `mapping dense error` terbukti bukan bottleneck utama. MHA dan FFN tetap mahal, tetapi masih jauh di bawah loss-gradient build pada workload ini.

## Tindak Lanjut

- Jika optimasi lanjutan diperlukan, fokus berikutnya seharusnya pada `loss gradient build`, kemungkinan lewat kernel native/sparse masked softmax-cross-entropy.
- `output dense forward` juga tetap mahal; optimasi projector atau kernel matmul khusus vocab besar bisa memberi dampak tambahan.
- Pertahankan `npm test` sebagai entry point resmi, tetapi gunakan `transformer_perf_breakdown.ts` untuk audit regresi performa transformer yang lebih fair.
- Jangan menghapus snapshot benchmark lama; gunakan snapshot ini sebagai baseline apple-to-apple baru untuk optimasi transformer berikutnya.

## Versioning

Versi aktif proyek saat ini adalah `1.3.1`.

Proyek ini memakai format versi `MAJOR.MINOR.PATCH` seperti `1.3.1`.

- Angka paling depan (`MAJOR`): perubahan besar yang biasanya membawa breaking change atau perubahan arsitektur utama.
- Angka tengah (`MINOR`): penambahan fitur baru atau peningkatan yang tetap kompatibel dengan versi sebelumnya.
- Angka paling belakang (`PATCH`): perbaikan bug, optimasi kecil, cleanup, atau perubahan minor yang tidak mengubah API utama.

Contoh:
- `1.3.1`: rilis mayor `1`, minor `3`, patch `1` untuk optimasi internal transformer.
- `1.1.4`: masih di mayor `1` dan minor `1`, tetapi sudah ada 4 patch/perbaikan kecil dari baseline `1.1.0`.
