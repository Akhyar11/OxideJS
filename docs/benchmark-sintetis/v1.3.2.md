## Perubahan Penting Dibanding Versi Sebelumnya

- Ditambahkan kernel native `masked sparse softmax-cross-entropy` untuk path training full-sequence transformer.
- `Transformers.backward()` sekarang memakai kernel native tersebut bila backend aktif, dengan fallback JS tetap tersedia dan diverifikasi setara pada correctness suite.
- Ditambahkan projector inference khusus LM untuk `forwardNextToken()` / `predict()`, sehingga last-token logits tidak lagi melalui gather buffer + dense generic path.
- Harness benchmark transformer sekarang mencatat `inferenceOnlyMsPerIter` agar jalur generation terpisah dari training.

## Interpretasi

- Benchmark `v1.3.1` tetap menjadi baseline before/after yang valid untuk command yang sama.
- Peningkatan paling meyakinkan terlihat pada harness end-to-end dataset (`msPerBatch` dan `samplesPerSec`), yang sekarang jauh lebih cepat setelah kernel native aktif.
- Microbenchmark `forwardOnly/backwardOnly/trainingStep` masih cukup noisy, jadi harus dibaca bersama profiling stage dan end-to-end benchmark, bukan sebagai satu-satunya sumber kebenaran.
- Tidak ada perubahan API publik. Semua perubahan bersifat internal dan terfokus pada performa.

## Tindak Lanjut

- Jika optimasi lanjutan diperlukan, fokus berikutnya adalah projector full-sequence vocab besar pada training path, karena `output dense forward` masih mahal.
- Pertimbangkan kernel native tambahan untuk full-sequence output projection atau fused projector+loss agar jalur training bisa turun lebih jauh.
- Pertahankan benchmark inference-only di `transformer_perf_breakdown.ts` agar perubahan generation tidak tertutup oleh noise training benchmark.

## Versioning

Versi aktif proyek saat ini adalah `1.3.2`.

Proyek ini memakai format versi `MAJOR.MINOR.PATCH` seperti `1.3.2`.

- Angka paling depan (`MAJOR`): perubahan besar yang biasanya membawa breaking change atau perubahan arsitektur utama.
- Angka tengah (`MINOR`): penambahan fitur baru atau peningkatan yang tetap kompatibel dengan versi sebelumnya.
- Angka paling belakang (`PATCH`): perbaikan bug, optimasi kecil, cleanup, atau perubahan minor yang tidak mengubah API utama.

Contoh:
- `1.3.2`: rilis mayor `1`, minor `3`, patch `2` untuk optimasi native transformer.
- `1.1.4`: masih di mayor `1` dan minor `1`, tetapi sudah ada 4 patch/perbaikan kecil dari baseline `1.1.0`.
