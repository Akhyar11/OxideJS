## Interpretasi

- Dibanding baseline immediate sebelum patch ini pada branch yang sama, peningkatannya ada tetapi inkremental.
- Perbandingan apple-to-apple terhadap baseline immediate:
  - `inferenceOnlyMsPerIter`: `114.78 -> 111.81` (`-2.6%`)
  - `forwardOnlyMsPerIter`: `299.97 -> 390.29` (`+30.1%`, lebih lambat)
  - `backwardOnlyMsPerIter`: `1242.50 -> 1331.23` (`+7.1%`, lebih lambat)
  - `trainingStepMsPerIter`: `1309.33 -> 1386.49` (`+5.9%`, lebih lambat)
  - `msPerBatch`: `743.25 -> 733.12` (`-1.4%`, lebih cepat)
  - `msPerSample`: `11.8330 -> 11.6718` (`-1.4%`, lebih cepat)
  - `samplesPerSec`: `84.51 -> 85.68` (`+1.4%`)
- Artinya:
  - patch ini tidak memberi kemenangan besar, tetapi ada perbaikan kecil yang konsisten pada benchmark end-to-end median
  - microbenchmark synthetic batch justru memburuk pada beberapa metrik, jadi optimasi ini kemungkinan lebih cocok untuk pola batch nyata benchmark resmi daripada synthetic batch terkontrol
  - smoke run `npm test` terbaru juga terlihat sehat di `94.78 samples/s`, `662.70 ms/batch`, `10.5507 ms/sample`, tetapi angka itu tetap saya perlakukan sebagai satu run noisy, bukan baseline utama

## Tindak Lanjut

- Gunakan `v2.0.1` sebagai catatan bahwa paralelisasi native loss kernel memberi kenaikan kecil pada benchmark end-to-end resmi, tetapi tidak memperbaiki semua microbenchmark.
- Optimasi berikutnya sebaiknya menargetkan projector full-sequence vocab besar atau mencoba heuristik threshold kapan kernel paralel diaktifkan agar synthetic batch dan dataset nyata sama-sama membaik.
- Pertahankan benchmark inference-only dan end-to-end bersama-sama agar optimasi yang bagus di microbench tidak salah dibaca sebagai kemenangan mutlak.

## Versioning

Versi aktif proyek saat ini adalah `2.0.1`.

Proyek ini memakai format versi `MAJOR.MINOR.PATCH` seperti `2.0.1`.

- Angka paling depan (`MAJOR`): perubahan besar yang biasanya membawa breaking change atau perubahan arsitektur utama.
- Angka tengah (`MINOR`): penambahan fitur baru atau peningkatan yang tetap kompatibel dengan versi sebelumnya.
- Angka paling belakang (`PATCH`): perbaikan bug, optimasi kecil, cleanup, atau perubahan minor yang tidak mengubah API utama.

Contoh:
- `2.0.1`: rilis mayor `2`, minor `0`, patch `1` untuk optimasi lanjutan native transformer.
- `1.1.4`: masih di mayor `1` dan minor `1`, tetapi sudah ada 4 patch/perbaikan kecil dari baseline `1.1.0`.
