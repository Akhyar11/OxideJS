## Perubahan Penting Dibanding Versi Sebelumnya

- `Dense.forward()` untuk activation linear tidak lagi melakukan copy penuh `z -> result`.
- `Dense.backward()` untuk activation linear tidak lagi melakukan perkalian elemen-demi-elemen dengan gradien identitas yang sebenarnya tidak perlu.
- `add_bias_native()` sekarang mengikuti layout row-major dan diparalelkan untuk matriks besar.
- Audit lanjutan pada inference path mencoba gather last-token kontigu lalu memproyeksikan dengan `Dense.forward()`, tetapi hasilnya lebih lambat dan sengaja tidak dipertahankan.
- Tidak ada perubahan API publik, objective training, causal mask, pad mask, maupun shape output.

## Interpretasi

- Dibanding baseline immediate `v2.0.1`, patch ini memberi kenaikan yang besar dan apple-to-apple pada workload yang sama.
- Perbandingan median terhadap `v2.0.1`:
  - `forwardOnlyMsPerIter`: `390.29 -> 236.56` (`-39.4%`)
  - `backwardOnlyMsPerIter`: `1331.23 -> 973.07` (`-26.9%`)
  - `trainingStepMsPerIter`: `1386.49 -> 1019.69` (`-26.5%`)
  - `msPerBatch`: `733.12 -> 394.12` (`-46.2%`)
  - `msPerSample`: `11.6718 -> 6.2747` (`-46.2%`)
  - `samplesPerSec`: `85.68 -> 159.37` (`+86.0%`)
- Inference-only tidak ikut membaik pada patch ini. Percobaan alternatif inference path yang lebih generik juga lebih lambat, jadi state final tetap memakai projector last-token khusus.
- Stage profile menunjukkan bottleneck forward yang lama di `output dense forward` turun drastis (`277.79 -> 44.87 ms` jika dibanding snapshot `v2.0.1`). Bottleneck terbesar yang tersisa sekarang adalah `loss gradient build`.
- Benchmark paling layak dijadikan acuan utama untuk patch ini adalah `transformer_perf_breakdown` median 3 run, bukan single-run `npm test`.

## Tindak Lanjut

- Audit khusus jalur inference last-token karena patch ini berfokus pada projector full-sequence training, bukan generation path.
- Optimasi berikutnya sebaiknya menargetkan `loss gradient build`, karena itu sekarang menjadi hotspot dominan setelah projector dipercepat.
- Pertahankan benchmark inference-only, training-step, dan end-to-end bersama-sama agar optimasi berikutnya tidak hanya menang pada satu sudut pandang.

## Versioning

Versi aktif proyek saat ini adalah `2.0.2`.

Proyek ini memakai format versi `MAJOR.MINOR.PATCH` seperti `2.0.2`.

- Angka paling depan (`MAJOR`): perubahan besar yang biasanya membawa breaking change atau perubahan arsitektur utama.
- Angka tengah (`MINOR`): penambahan fitur baru atau peningkatan yang tetap kompatibel dengan versi sebelumnya.
- Angka paling belakang (`PATCH`): perbaikan bug, optimasi kecil, cleanup, atau perubahan minor yang tidak mengubah API utama.

Contoh:
- `2.0.2`: rilis mayor `2`, minor `0`, patch `2` untuk optimasi projector transformer tanpa perubahan API.
- `1.1.4`: masih di mayor `1` dan minor `1`, tetapi sudah ada 4 patch/perbaikan kecil dari baseline `1.1.0`.
