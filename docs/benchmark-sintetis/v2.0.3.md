## Benchmark Gagal

## Perubahan Penting Dibanding Versi Sebelumnya

- Kernel native `maskedSparseSoftmaxCrossEntropyInto` diubah dari pemrosesan per-token ke pemrosesan per token-block agar traversal `logits` dan `grad` lebih cache-friendly pada layout `[vocab, totalTokens]`.
- Penulisan nol untuk posisi invalid sekarang diintegrasikan ke pass block yang sama, bukan loop invalid terpisah per token.
- `MultiHeadAttention.backward()` tidak lagi menyalin bobot `q/k/v` ke buffer lama hanya untuk menghitung `gradInput`; perhitungan `gradInput` dipindah sebelum update optimizer sehingga semantics tetap sama tanpa overhead copy matriks.
- Tidak ada perubahan API publik, objective training, target shape `[seqLen, batch]`, maupun inference path last-token.

## Interpretasi

- Patch ini jelas menargetkan bottleneck yang tepat: `loss gradient build` turun dari `263.37 ms` menjadi `64.85 ms` pada median stage profile.
- Karena hotspot dominan berkurang besar, metrik end-to-end ikut naik signifikan meskipun forward path praktis datar.
- `forwardOnlyMsPerIter` sedikit naik, tetapi trade-off itu tertutup jauh oleh penurunan besar pada path backward/training-step.
- Benchmark acuan utama tetap `transformer_perf_breakdown` median 3 run karena ia memisahkan inference, forward, backward, training-step, dan stage profile.

## Risiko Residual

- `output dense backward` masih mahal dan sekarang menjadi salah satu hotspot utama berikutnya.
- `MHA backward` masih cukup besar karena pasca-kernel attention masih memerlukan beberapa GEMM untuk proyeksi grad dan akumulasi `gradInput`.
- Forward path belum disentuh secara agresif pada patch ini, sehingga ruang optimasi berikutnya ada pada projector backward dan fusion MHA projection.

## Tindak Lanjut

- Audit fast path native khusus projector backward linear dengan vocab besar.
- Evaluasi fusion tambahan di `MHA.backward` jika profil berikutnya menunjukkan biaya GEMM pasca-attention tetap dominan.
- Pertahankan benchmark command dan konfigurasi yang sama agar histori tetap comparable.

## Versioning
Versi aktif proyek saat ini adalah `2.0.2`.

Proyek ini memakai format versi `MAJOR.MINOR.PATCH` seperti `2.0.2`.

- Angka paling depan (`MAJOR`): perubahan besar yang biasanya membawa breaking change atau perubahan arsitektur utama.
- Angka tengah (`MINOR`): penambahan fitur baru atau peningkatan yang tetap kompatibel dengan versi sebelumnya.
- Angka paling belakang (`PATCH`): perbaikan bug, optimasi kecil, cleanup, atau perubahan minor yang tidak mengubah API utama.

Contoh:
- `2.0.2`: rilis mayor `2`, minor `0`, patch `2` untuk state repo saat ini.
- `2.0.3`: kandidat patch berikutnya untuk optimasi runtime transformer tanpa perubahan API publik.
