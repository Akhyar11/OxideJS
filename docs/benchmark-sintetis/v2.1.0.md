## Interpretasi

- Scaling depth relatif mendekati linear tetapi tidak persis linear.
- `loss gradient build` tidak tumbuh dominan terhadap depth; biaya utama bertambah pada stack block itu sendiri, terutama pasangan `MHA forward/backward` dan `FFN forward/backward`.
- Untuk konfigurasi benchmark ini, `numBlocks=2` masih memberi throughput yang cukup baik.
- `numBlocks=4` dan terutama `6` menurunkan throughput end-to-end cukup tajam pada CPU ini, jadi kedalaman lebih besar perlu dibaca sebagai tradeoff kapasitas vs latency/training cost.

## Risiko Residual

- Benchmark ini belum mengukur kualitas model, hanya biaya runtime.
- Profil per-stage agregat masih menjumlah lintas block; belum ada label stage per-index block.
- Jika depth lebih tinggi dari `6`, bottleneck CPU memory bandwidth dan GEMM kecil kemungkinan akan makin terlihat.

## Tindak Lanjut

- Tambahkan benchmark kualitas untuk melihat apakah `4` atau `6` block memberi gain yang sepadan terhadap biaya runtime.
- Jika ingin profiling lebih tajam, pecah label profiler menjadi `MHA forward [block i]`, `FFN backward [block i]`, dan seterusnya.
- Evaluasi fast path native khusus projector backward jika target deployment tetap CPU-bound.

## Versioning
Versi aktif proyek saat ini adalah `2.1.0`.

Proyek ini memakai format versi `MAJOR.MINOR.PATCH` seperti `2.1.0`.

- Angka paling depan (`MAJOR`): perubahan besar yang biasanya membawa breaking change atau perubahan arsitektur utama.
- Angka tengah (`MINOR`): penambahan fitur baru atau peningkatan yang tetap kompatibel dengan versi sebelumnya.
- Angka paling belakang (`PATCH`): perbaikan bug, optimasi kecil, cleanup, atau perubahan minor yang tidak mengubah API utama.

Contoh:
- `2.1.0`: rilis mayor `2`, minor `1`, patch `0` untuk fitur/kapabilitas baru pada transformer multi-block dan benchmark scaling.
- `2.0.3`: rilis mayor `2`, minor `0`, patch `3` untuk optimasi runtime tanpa perubahan capability model.
