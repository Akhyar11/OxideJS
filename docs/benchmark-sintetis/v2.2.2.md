## Interpretasi

- Pada workload ini, `RNN` tetap menjadi recurrent tercepat, disusul `GRU`, lalu `LSTM`.
- Untuk transformer, `next-token` legacy path lebih ringan daripada `full-token`, sesuai jumlah logits/loss yang diproses.
- `full-token-trimPad` pada snapshot ini tidak lebih cepat daripada `full-token-padded-no-trim`; ini menunjukkan harness padded kecil saat ini belum cukup besar/stabil untuk menonjolkan keuntungan trimming di benchmark mikro singkat.
- Acuan utama snapshot ini adalah `suite_gabungan` karena paling representatif terhadap command resmi repo (`npm test`).

## Versioning
Versi aktif proyek saat ini adalah `2.2.2`.

Proyek ini memakai format versi `MAJOR.MINOR.PATCH` seperti `2.2.2`.

- Angka paling depan (`MAJOR`): perubahan besar yang biasanya membawa breaking change atau perubahan arsitektur utama.
- Angka tengah (`MINOR`): penambahan fitur baru atau peningkatan yang tetap kompatibel dengan versi sebelumnya.
- Angka paling belakang (`PATCH`): perbaikan bug, optimasi kecil, cleanup, atau perubahan minor yang tidak mengubah API utama.

Contoh:
- `2.2.0`: rilis minor `2` untuk fitur dynamic padding (`trimPadding`) dan proyek `math-reasoning-ai`.
- `2.2.1`: patch snapshot lokal untuk optimasi buffer keluarga recurrent dan benchmark pendampingnya.
- `2.2.2`: patch untuk suite gabungan root, benchmark family model, dan correctness learning suite.
