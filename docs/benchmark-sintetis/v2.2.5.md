## Benchmark Gagal

## Perubahan Penting Dibanding Versi Sebelumnya (`v2.2.4`)

- `Sequential.resolveLossName()` tidak lagi memanggil `save()` pada layer output.
- Validation generic `Sequential` mendukung batching untuk model non-recurrent.
- Loss full-sequence transformer memakai native masked sparse loss atau log-sum-exp langsung dari logits tanpa softmax penuh.
- `Embedding` memakai typed token buffer dan menggabungkan reorder + validasi token dalam satu pass.
- Tokenizer BPE memakai cache encode dan merge compact in-place untuk training/update.

## Interpretasi

- Snapshot ini menunjukkan suite benchmark tetap `success` setelah patch performa internal.
- Angka transformer `full-token` pada run ini lebih cepat daripada snapshot `v2.2.4`, tetapi benchmark sintetis tetap sensitif terhadap variasi runtime. Perbandingan performa final sebaiknya memakai median beberapa run.
- Optimasi tokenizer BPE belum punya benchmark micro khusus di suite root, sehingga dampak performanya belum terukur di tabel ini.

## Tindak Lanjut

- Tambahkan benchmark micro untuk `BPETokenizer.train()` dan `BPETokenizer.update()` pada corpus besar.
- Pertimbangkan snapshot multi-run median untuk benchmark transformer/recurrent.

## Versioning

Versi aktif proyek saat ini adalah `2.2.5`.

Proyek ini memakai format versi `MAJOR.MINOR.PATCH` seperti `2.2.5`.

- Angka paling depan (`MAJOR`): perubahan besar yang biasanya membawa breaking change atau perubahan arsitektur utama.
- Angka tengah (`MINOR`): penambahan fitur baru atau peningkatan yang tetap kompatibel dengan versi sebelumnya.
- Angka paling belakang (`PATCH`): perbaikan bug, optimasi kecil, cleanup, atau perubahan minor yang tidak mengubah API utama.

Contoh:
- `2.2.4`: patch untuk ergonomi API `predictMode`, sinkronisasi docs, dan refactor correctness suite.
- `2.2.5`: patch untuk optimasi hot path training/validation, embedding, dan tokenizer BPE.
